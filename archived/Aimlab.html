Aimlab 2.0

Overview:

This was a fun project I did with a friend to replicate the game “Aimlab”. The game is designated to improve your skill
in first-person shooters. The concept is simple. Configure your settings, click on blue balls as fast and as accurately
as you can. Our version was to attempt to replicate the clicking and score-keeping aspect of the game as close as we could.

Contribution:

We followed this API which locks your mouse when clicking on the canvas. It will then make canvas full screen.
We used this to simulate a first-person view. We had to create our own sprites, so took screenshots from a few games to
represent these and the backgrounds.

We created a coordinate system based off the placement of the mouse. Each sprite had a coordinate and did not move. The only
sprites that moved were the crosshair and gun, even though those were the two that seemingly weren’t moving. We did this by
always changing their coordinates in alignment with the mouse’s movement.

Action listeners were set in place to check for if the user clicks the mouse. That moment the mouse is clicked, the gun’s animation
will go off, and the gun sound is played. Additionally, if the crosshair’s coordinates overlap with a ball’s coordinates, points
will be added, and a popping sound is played. Lastly, if there is no overlap, it’s a miss, and points will be deducted. This continues
for 60 seconds until the game is over.

Complications:

- Gun sprite animation. As you click more, the gun’s animation speeds up, I would’ve liked to have fixed this, but was
unsure of how to fix it considering how we implemented the game.

- We tried to create a 3D space in 2D. We thought we could sell the 3D effect, but one of our ultimate downfalls was
the background. We tried many ideas, however ultimately, we had to settle with a static 2D image.

- General implementation. We started with a JavaScript-based game engine. However, I think we misunderstood how to use
this tool and rearranged/mishandled a lot of different features to try and meet our wants.

Bokjab Chat

Overview:

Bokjab Chat was one of the last final projects I did during college.
The goal was to create an Android application that allowed you to send friend requests
and chat with others that were using the same app.
Additionally, our professor requested that we have a page dedicated, dynamic weather page.
We designed the whole front end with Android Studio (majority Java, some xml), used mainly
JavaScript, and a bit of HTML and SQL with our backend.
Finally, we used Heroku as our connecting cloud service.

Contributions:

While I was not the main one assigned to it, I did help with the sign in process we implemented.
I helped figure out how to retrieve input from the user and checking the credentials.

My focus of this project was to figure all the ins and outs of the weather portion of the app.
I was able to achieve this by using OpenWeatherMap’s Weather API.
By making a request with the API, you can achieve several different types of weather forecasts
for a desired area (we made use of the 5-day and 24-hour requests).

The app will automatically request you turn location services on, and by doing this it will show your
locations current weather. If you decline, we checked with the professor, and we all decided that it will
default to Seattle. You can also change the location by typing in a new zip code.

In the middle is a recycler viewer. Which allows you to scroll and view the current 24-hour forecast. Above and below
are standard image and text views, which will also adjust according to the current location’s weather.

Lastly, the home page will also display weather information. This is determined by your default location, then your last searched location.
